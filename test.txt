Room Tone — Google Sheets Integration
Overview
The Room Tone website is a static site (HTML/CSS/JS, no build step) for a vintage furniture shop. Currently all product data is hardcoded in shop.html and product.html. The goal is to make the shop dynamically pull product data from a public Google Sheet so that adding/editing inventory only requires updating the spreadsheet and dropping images into the repo.
Google Sheet Structure
The sheet has these columns (row 1 headers):
| ITEM NAME | PRICE | TYPE | DESCRIPTION | DIMENSIONS | CONDITION | NUMBER OF IMAGES |

Here is the link to the sheet: https://docs.google.com/spreadsheets/d/1R9MqsYFdHzhx3grz13PmSpQDuNN-l6r594mTB2OC8_Y/edit?gid=0#gid=0

ITEM NAME: Display name of the product (e.g. "Lounge Chair")
PRICE: Numeric price (e.g. 1500)
TYPE: Category for filtering. Must match one of the existing sidebar categories: tables, seating, lighting, storage, textiles, art, objects
DESCRIPTION: Freeform product description text
DIMENSIONS: Freeform dimensions text
CONDITION: Freeform condition description
NUMBER OF IMAGES: Integer. How many images exist in the repo for this item

Slug Convention
The slug is derived from ITEM NAME by lowercasing, replacing all non-alphanumeric characters with hyphens, collapsing multiple hyphens, and trimming leading/trailing hyphens.
Examples:

"Lounge Chair" → lounge-chair
"Art + Mirrors" → art-mirrors
"Coffee Table" → coffee-table

Image Convention
Product images live in Items/ and follow this naming pattern:
Items/{slug}-1.webp
Items/{slug}-2.webp
…
Items/{slug}-N.webp
Where N matches the NUMBER OF IMAGES column. The first image (-1.webp) is used as the shop grid thumbnail.
Data Fetching
Create js/sheets.js — a vanilla JS module (no dependencies, no build step) that:

Fetches from the Google Sheets visualization API endpoint:

   https://docs.google.com/spreadsheets/d/{1R9MqsYFdHzhx3grz13PmSpQDuNN-l6r594mTB2OC8_Y}/gviz/tq?tqx=out:json&sheet={Roomtone Items}

Has SPREADSHEET_ID and SHEET_NAME as config constants at the top of the file (easy to find and edit). Default SHEET_NAME to "Sheet1".
Parses the Google Visualization JSON response. Note: Google wraps the JSON in google.visualization.Query.setResponse(…); — strip the wrapper to get the inner JSON object.
Maps column headers (case-insensitive) to internal keys: name, price, type, description, dimensions, condition, imageCount.
For each row, derives the slug from the name, builds an array of image paths based on imageCount, and normalizes type to lowercase.
Skips empty rows (no name).
Exposes two functions:

fetchProducts() → returns a Promise resolving to an array of product objects
fetchProductBySlug(slug) → returns a Promise resolving to a single product or null



Each product object shape:
js{
  name: "Lounge Chair",
  slug: "lounge-chair",
  price: 1500,
  type: "seating",
  description: "Beech bentwood frame with…",
  dimensions: "H 39.5″ × W 25″ × …",
  condition: "Wear consistent with age…",
  imageCount: 6,
  images: ["Items/lounge-chair-1.webp", "Items/lounge-chair-2.webp", …]
}
shop.html Changes
Replace the hardcoded <section class="product-grid"> contents. On page load:

Read the ?cat= query param for category filtering (default "all").
Highlight the active .cat-filter link in the sidebar (same as current behavior).
Call fetchProducts(), then for each product:

Skip if cat !== 'all' and product.type !== cat
Create an <a> element with class product-card, data-category set to product.type
Link to product.html?item={slug}
Inner HTML: product thumbnail image (first image path) + <p class="product-info"> with lowercase name and price


Show a "loading…" message while fetching, and a graceful error message if the fetch fails.
Keep all existing sidebar HTML, nav structure, CSS classes, and hamburger button exactly as they are.

product.html Changes
Replace the hardcoded product detail. On page load:

Read ?item= query param to get the slug. Redirect to shop.html if missing.
Call fetchProductBySlug(slug).
If not found, show "item not found."
If found, update document.title to "{name} — Room Tone" and render:

Image carousel (.product-images-col): main image with prev/next buttons (only if >1 image), thumbnail strip below (only if >1 image). Same HTML structure and CSS classes as the current product.html.
Info column (.product-info-col): name + price, then description/dimensions/condition sections (only render each section if the field is non-empty), then the "email to purchase" line.


Initialize carousel JS inline (prev/next cycling, thumbnail click to select). The carousel logic should be self-contained in this file since it depends on the dynamically loaded image array. The separate carousel.js file is no longer needed for this page.
Show "loading…" while fetching, graceful error on failure.
Keep all existing sidebar HTML, nav structure, CSS classes, and hamburger button exactly as they are.

Important Constraints

No build step. No npm, no bundler, no framework. Vanilla HTML/CSS/JS only.
No external dependencies. Just fetch() and DOM APIs.
Preserve all existing CSS classes and HTML structure so that the current stylesheet continues to work without modification.
The sidebar, nav, category filters, hamburger menu — all stay exactly as they are. Only the main content area changes.
Use the IIFE pattern ((function() { … })()) to avoid polluting global scope, except for the RoomToneSheets module which should be a global (or namespaced however you prefer, as long as both pages can access it).
Add a small .loading-msg style if needed (just italic body font, centered) but try to avoid CSS changes.

Setup Instructions (for the README or a comment)
After implementing, the user needs to:

Create the Google Sheet with the column headers above.
File → Share → Publish to web (entire document).
Copy the spreadsheet ID from the URL and paste it into js/sheets.js where SPREADSHEET_ID is defined.
Name product images following the {slug}-N.webp convention in Items/.
That's it — the site reads from the sheet on every page load.
